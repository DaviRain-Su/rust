# 错误处理

rust对可靠行的执着延伸到了错误处理，错误对于软件来说是不可避免的，所以rust有很多特性来处理错误的情况。

很多情况下，rust要求你承认出错误的可能性并在编译代码之前就采取行动。通过确保不会只有在将代码部署到生产之后才会发现错误来使程序更可靠。

rust将错误组合成两个主要类别：可恢复错误和不可恢复错误，可恢复错误通常代表向用户报告错误和重试操作是合理的情况。比如未找到文件，不可恢复错误通常是bug的同义词，比如尝试访问超过数组结尾的位置。

对于可恢复错误有Result<T, E>，以及panic!, 他在遇到不可恢复错误时停止程序运行。



## panic！与不可恢复的错误

rust有Panic！当执行这个宏时，程序会打印出一个错误信息，展开并清理栈数据，然后退出。出现这种情况的场景通常是检测到一些类型的bug而且程序并不清楚该如何处理它。


panic中的栈展开与终止

当出现panic时，程序默认会开始展开，这意味这rust会回溯栈并清理它遇到的每一个函数的数据，不过这个回溯并清理的过程有很多工作。 另一种选择是直接终止，这不会清理数据就退出程序，那么程序所使用的内存需要由操作系统来清理，如果你需要项目的最终二进制文件越小越好，可以由panic时展开切换为终止。通过cargo.toml的 [profile]部分增加   panic = 'abort';

    [profile.release]
    panic = 'abort'

    fn main() {
        panic!("crash and burn");
    }

## 使用panic！的backtrace

另一个因为我们代码中的bug引起别的库中的panic！的例子，而不是直接使用。

    fn main() {
        let v = vec![1, 2, 3];
        
        v[99];
    }

这样会得到任何对应vector中这个元素的内存位置的值，甚至是这些内存并不属于vector的情况。这被称为缓冲区溢出。并可能导致安全漏洞。

为来使程序远离这些漏洞，如果尝试读取一个索引不存在的元素，rust会停止执行并拒绝继续。

设置RUST_BACKTRACE环境变量来得到一个backtrace backtrace是一个执行到目前位置所有被调用的函数列表。

阅读backtrace的关键是从头开始直到读到发你你编写的文件。

    $ RUST_BACKTRACE = 1 cargo run
为来获取带有这些信息的backtrace，必须启动debug标示。

## Ruslt与可恢复的错误

大部分的错误并没有严重到需要程序完全停止执行。有时，一个函数会因为一个容易理解并作出反应的原因失败。

例如尝试打开一个文件不过由于文件并不存在而操作失败，这时我们可能想要创建这个文件而不是终止进程。

    enum Result<T, E> {
        Ok(T), 
        Err(E),
    }

T 和 E 是泛型类型参数； T代表成功时会返回OK成员中的数据类型，而E代表失败时会返回Err成员的错误类型。

因为Result有这些泛型类型参数，我们可以将ruslt类型和标准库中为其定义的函数用于很多不同的场景，这些情况中需要返回的成功值和失败值可能会各不相同。

    use std::fs::File;

    fn main() {
        let f = File::open("hello.txt");
    }

File::open 函数的返回值类型是Result<T, E>. 这里的泛型参数T放入来成功值的类型 std::fs::FIle,它是一个文件句柄。E被用在失败值上E的类型是std::io::Error;

这个返回值类型说明File::open调用可能会成功并返回一个可以进行读写的文件句柄。这个函数也可能会失败；例如文件可能并不存在，或者可能没有访问文件的权限。

File::open 需要一个方式告诉我是成功还是失败，并同时提供给我文件句柄或错误信息。
而这些信息正是Reuslt枚举可以提供的。

当File::open成功的情况下，变量f的值将会是一个包含文件句柄的OK实例，在失败的情况下，f的值会是一个包含更多关于出现来何种错误的Err实例。

    use std::fs::File;

    fn main() {
        let f = File::open("hello.txt");

        let f = match f {
            Ok(file) => file,
            Err(error) => {
                panic!("There was a problem opening the file : {:?}", error)
            },
        };
    }


当结果OK时，返回OK成员中的file值，然后将这个文件句柄赋值给变量f。match之后，可以利用这个文件句柄进行读写。
